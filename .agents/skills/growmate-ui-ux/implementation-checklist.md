# GrowMate UI/UX 実装チェックリスト

**コーディング中・実装完了直前**の自己確認用。本 Skill の主用途はレビューではなく **開発時の判断材料** である。PR レビューや品質ゲートは `quality-gate` に委譲する。

## A. 実装前（着手前に確認）

- [ ] 対象機能の `docs/plans/*` / `docs/specs/*` を読んだ
- [ ] 同種画面の既存パターンを grep で確認した（`/setup/*`, ダッシュボード等）
- [ ] 中〜大規模変更なら UI たたき台を合意済み（§1.8）
- [ ] 空状態・ローディング・エラー・成功の 4 状態を設計に含めた

## B. コーディング中（GrowMate 固有）

- [ ] shadcn/ui コンポーネントを再利用している（独自 primitives を増やしていない）
- [ ] 色は `app/globals.css` のセマンティックトークン（生の `bg-*-600` 乱用なし）
- [ ] 1 画面の主 CTA が 1 つ
- [ ] 専門用語に初出補足がある。ユーザー向け英語略語なし
- [ ] フォームに visible label がある（placeholder のみ禁止）
- [ ] AI 処理 300ms 超に loading / skeleton がある
- [ ] エラーメッセージに **原因と復帰手順** がある
- [ ] 破壊的操作に確認ダイアログがある
- [ ] モーダル on モーダルになっていない

## C. a11y・操作（開発時に組み込む）

- [ ] `focus-visible` リング
- [ ] アイコンのみボタンに `aria-label`
- [ ] 非同期 submit 中はボタン disabled + スピナー
- [ ] タップ/クリック領域が十分（目安 44×44px）

## D. 採用しない（開発中に提案されても却下）

- ~~新カラーパレット / Google Fonts 差し替え~~
- ~~glassmorphism / bento 等のスタイル刷新~~
- ~~オンボーディング UI の Skill 内判断~~（→ `docs/plans/*`）
